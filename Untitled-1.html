About This Project

Built for the AIML Era

Project Title: Smart Content Discovery and Recommendation Engine using Content-Based Filtering and TMDB API.

With the exponential rise of OTT platforms, users face the "paradox of choice." This platform solves that by using a Content-Based Filtering approach — filtering by genre, cast, language, and release era to deliver hyper-personalized recommendations in real-time.

Unlike Netflix or Prime's black-box algorithms, CineAI is platform-agnostic — it shows you what's available across ALL services, not just one.

🎯
Content-Based Filtering
Recommends based on item attributes (cast, genre, era) — no prior user history needed. Solves the Cold Start problem.
⚡
Real-Time TMDB API
Fetches up-to-date metadata, posters, and OTT streaming availability from The Movie Database API.
🖥️
Single Page Application UX
Dynamic modal-based interaction with no page reloads — reducing latency and cognitive friction for users.
📊
Multi-Variable Filtering
Users combine Type + Language + Genre + Era + Cast + Platform to get surgical, precise recommendations.
How It Works
The Recommendation Pipeline
1
User Input
User selects content type, language, genre, era, and cast preferences via the dynamic form.
2
Query Builder
The system constructs a structured query from multi-variable inputs using Content-Based Filtering logic.
3
TMDB API Call
Real-time request to TMDB API fetches matching movies with posters, metadata, and OTT availability.
4
Ranking & Filter
Results are ranked by relevance score, user rating, and preference match using cosine similarity.
5
Display
Final recommendations rendered in a Netflix-style grid with modal detail view — no page reload.
Technology
Tech Stack Used
🎬
TMDB API
The Movie Database API provides real-time movie metadata, posters, cast info, and OTT streaming availability across 40+ platforms.
⚛️
Frontend (HTML/CSS/JS)
Single Page Application architecture with AJAX-driven dynamic modals. No full page reloads — smooth, app-like experience.
🧠
Content-Based Filtering
Algorithm based on Pazzani & Billsus (2007) — item attribute matching using cosine similarity over genre, cast, and keyword vectors.
🐍
Python (Backend)
Flask/FastAPI backend processes filter queries, calls TMDB, and applies the recommendation logic before returning JSON to frontend.
📊
Data Analysis
Pandas and NumPy used to process large volumes of movie metadata, extract patterns, and clean/normalize dataset features.
☁️
Real-Time Integration
Live TMDB API integration with caching layer to ensure fast response times while providing always-fresh movie data.
